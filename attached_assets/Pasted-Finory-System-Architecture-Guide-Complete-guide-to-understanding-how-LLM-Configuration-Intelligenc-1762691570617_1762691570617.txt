What is it?

Three Key Concepts:
1ï¸âƒ£ LLM Providers

2ï¸âƒ£ Models
What: Specific AI brains from providers

Claude-3.5 (great at analysis)
Purpose: Different AI models have different strengths

Attributes:

Name: "gpt-4o", "gpt-5"
Type: Chat, Completion, Embedding
Context Window: How much text it can "remember" (e.g., 128k tokens)
Cost: Price per 1M tokens
Example:

Provider: OpenAI
Context: 128,000 tokens
Best for: Complex reasoning, module generation
Cost: $2.50 per 1M input tokens

3ï¸âƒ£ Agent Configurations
What: Pre-configured AI personalities/roles for specific tasks

Purpose: Different jobs need different AI behaviors

Built-in Agent Configs:

Builder - Understands ERP requirements, builds modules
Coordinator - Orchestrates other agents
Analyst - Analyzes data and processes
SME (Subject Matter Expert) - Domain expertise
Reviewer - Quality checks
Strategy - High-level planning
Structure:

{
  "role": "builder",
  "modelId": "gpt-5",
  "systemPrompt": "You are an expert ERP module builder...",
  "temperature": 0.7,
  "maxTokens": 4000
}

Why separate configs?

A Builder agent needs creativity (higher temperature)
A Reviewer agent needs precision (lower temperature)
Each uses the same model but with different "personalities"

FAQ
Q: Do I need to understand LLM Configuration to use Module Forge? A: No! As long as an admin has configured it once, Module Forge will work automatically.

Q: Can I use the same agent config for multiple agents? A: Yes! You can have 10 agents all using the "Builder" config, each with different names/purposes.

Q: What's the difference between Module Forge and Intelligence Forge? A:

Module Forge = Build ERP modules (Purchase Orders, Invoices)
Intelligence Forge = Build AI agents (assistants to help users)
Q: Where do I see modules I've created? A: Build Mode â†’ Modules (to edit), ERP Mode â†’ Use them operationally

Q: Can I customize AI agent behavior? A: Yes! Edit the agent config's system prompt, temperature, and max tokens in LLM Configuration.
